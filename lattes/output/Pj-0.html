Descrição: Estudos anteriores demonstram que médicos requerem informações sobre pacientes (para tomada de decisão) em diferentes localizações e situações. 2.   2008-2009. Processamento de Imagens de Cromossomos
Descrição: Um dos grandes problemas existentes na área de Bioinformática é a aquisição de processamento de imagens obtidas através de microscópios, como o caso das imagens de cromossomos, devido a sua qualidade e resolução. O principal objetivo deste trabalho é apresentar uma ferramenta de processamento de imagens capaz de processar as imagens de cromossomos adquiridas, efetuando a classificação automática dos cromossomos (cariotipagem). A idéia do projeto é o desenvolvimento de uma ferramenta em Java que possibilite destacar os cromossomos na imagem, separá-los e em seguida fazer sua cariotipagem. O estudo de processamento de imagens abrangiu as áreas de Realce de Imagens, sendo necessário o estudo das teorias matemáticas de Transformadas de Fourier e de Convolução. Dessa forma, foi possível o entendimento de como são utilizados filtros de passa baixa e alta em imagens, estes últimos responsáveis pelo realce das fronteiras dos objetos na imagem.. Situação: Concluído; Natureza: Pesquisa. Alunos envolvidos: Graduação: (2) / Doutorado: (1) . Descrição: O presente projeto tem como objetivo principal o uso da norma ISO 13250 Topic Maps para a organização e representação do conhecimento encontrado no PubMed. 1.   2006-2007. Desenvolvimento de um sistema para e-Learning
Descrição: O presente projeto destina-se à construção do protótipo de um sistema para apoio à produção de conteúdos para E-Learning. Efetivamente, pretende-se analisar com cuidado os produtos mais utilizados para implementação de plataformas E-Learning para constatar as que fazem praticamente tudo, ou seja, desde a gestão do processo do aluno à análise da freqüência e assiduidade deste. No entanto, no que diz respeito a conteúdos, permitem ao docente publicar aquilo que este bem entender, não dando qualquer suporte à produção do conteúdo propriamente dito. Foi para preencher este vazio que este sistema foi pensado. De momento, o sistema corresponde à integração de cinco componentes que foram desenvolvidos separadamente: três aplicações de produção de conteúdos (uma para provas, uma para aulas práticas/laboratoriais e uma para apresentações multimídia), um componente de integração de conteúdos, e um componente de gestão de atualizações. Os professores que trabalharão com o sistema editarão seus materiais didáticos em editores personalizados, especificamente criados para cada tipo de material. Estes documentos serão armazenados em formato XML, um formato próprio para o intercâmbio entre aplicações e processamento de documentos. A partir do conjunto de documentos XML gerados pelos editores, o sistema de e-Learning será capaz de gerar um conjunto de páginas Web referente à disciplina em questão, contendo páginas HTML e documentos PDF, Word, RDF, PPT e LaTeX, por exemplo. A partir deste portal Web gerado, os alunos terão acesso a todo o conteúdo da disciplina, conteúdo este organizado através de uma ontologia sobre a disciplina em questão (representação do conhecimento) definida pelo próprio professor.. 2.   2006-2007. Representação espaço-temporal utilizando SGBD Open-Source
Descrição: O objetivo principal da pesquisa a ser desenvolvida é a concepção e implementação de um modelo de dados para a representação de objetos com características espaciais e temporais utilizando, para tal, somente bancos de dados open-source. No desenvolvimento será utilizado o PostgreSQL, pelo fato do mesmo apresentar a extensão para tratamento de dados espaciais PostGIS. Já as características temporais terão de ser desenvolvidas utilizando-se as funcionalidades apresentadas pelo SGBD. Posteriormente, terão de ser combinadas, em um modelo de dados a ser desenvolvido, a implementação destas duas características. Os objetivos específicos da pesquisa em desenvolvimento são: * Compreender e utilizar o PostGIS sobre o PostgreSQL; * Implementar a representação temporal dos dados no PostgreSQL; * Combinar as representações espacial e temporal no PostgreSQL; * Validar, através de um estudo de caso, a solução desenvolvida para a representação espaço-temporal utilizando o PostgreSQL. .
